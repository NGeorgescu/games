How the computer solves it — Knuth's minimax

The solver follows the strategy Donald Knuth published in 1977. It never needs more than 5 guesses for any of the 1296 possible codes (here it averages about 4.48). The Hint button in "You crack it" mode teaches a different, by-hand method — differential probing — rather than this exhaustive search.

  1. Open with 1122. Every game starts with this same two-color guess — Knuth showed it leads to the best worst case.
  2. Keep the still-possible codes. The solver holds the set S of every code that fits all the feedback so far. It starts at all 1296 codes, and each guess's black/white pegs prune the ones that don't match.
  3. Score every candidate by its worst case. To pick the next guess it tries all 1296 codes as a candidate. For each, it groups the codes in S by the feedback that candidate would produce, and looks at the largest group — the most codes that could still survive that guess.
  4. Play the minimax guess. It picks the candidate whose largest group is the smallest — minimizing the worst case (that's the "minimax"). Ties go to a guess that's still a possible code.

Because it minimizes the worst case, no single lucky near-miss helps much — progress comes from guesses that split the remaining possibilities as evenly as possible.
